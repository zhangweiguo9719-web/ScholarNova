## 2. 模块职责说明

### 2.1 用户层

**React 前端应用**

| 模块 | 职责 | 关键技术 |
|------|------|----------|
| 搜索页面 | 接收用户查询，展示搜索结果 | React, Zustand |
| 论文详情 | 展示论文详细信息和 AI 分析 | React Router |
| 对比分析 | 多篇论文对比展示 | 表格组件 |
| 推荐页面 | 个性化推荐结果展示 | 反馈机制 |
| 设置页面 | 模型配置和系统设置 | 表单组件 |

### 2.2 API 网关层

**Nginx 反向代理**

| 功能 | 说明 |
|------|------|
| 负载均衡 | 分发请求到多个后端实例 |
| SSL 终止 | 处理 HTTPS 加密解密 |
| 静态缓存 | 缓存前端静态资源 |
| 请求限流 | 防止恶意请求和 DDoS |

### 2.3 应用层

#### 2.3.1 API 路由层

| 路由模块 | 职责 | 主要端点 |
|----------|------|----------|
| 搜索 API | 处理搜索请求 | POST /api/v1/search |
| 论文 API | 论文 CRUD 操作 | GET /api/v1/papers/{id} |
| 分析 API | 论文分析和对比 | POST /api/v1/papers/{id}/analyze |
| 推荐 API | 推荐和反馈 | POST /api/v1/recommendations |
| 模型 API | 模型配置管理 | POST /api/v1/model/config |

#### 2.3.2 业务逻辑层

| 模块 | 职责 | 关键能力 |
|------|------|----------|
| 查询规划器 | 解析用户查询，生成子查询 | LLM 查询理解、约束提取 |
| 检索排序器 | 多源检索、去重、排序 | 并行检索、智能去重、相关性排序 |
| 证据验证器 | 提取和验证证据 | 全文分析、证据定位、置信度评估 |
| 推荐引擎 | 个性化推荐 | 协同过滤、内容推荐、反馈学习 |
| LLM 网关 | 统一 LLM 调用接口 | 多模型支持、负载均衡、容错 |

#### 2.3.3 数据访问层

| 模块 | 职责 | 技术实现 |
|------|------|----------|
| ORM 模型 | 数据库对象映射 | SQLAlchemy (async) |
| 缓存管理 | Redis 缓存操作 | redis-py |
| 消息队列 | 异步任务处理 | Redis Pub/Sub |
| 文件存储 | PDF 等文件管理 | 本地文件系统 |

### 2.4 数据源层

#### 2.4.1 学术数据源

| 数据源 | 覆盖领域 | API 限制 | 特点 |
|--------|----------|----------|------|
| Semantic Scholar | 计算机科学、生物医学 | 100 req/5min | 数据质量高，引用数据全 |
| OpenAlex | 全学科 | 100K req/day | 开放数据，覆盖范围广 |
| Crossref | 全学科 | 50 req/sec | DOI 解析，元数据标准 |
| arXiv | 预印本 | 3 req/sec | 最新研究，开放获取 |

#### 2.4.2 LLM 服务

| 提供商 | 模型 | 特点 |
|--------|------|------|
| OpenAI | GPT-4o, GPT-4 | 性能强，成本较高 |
| Anthropic | Claude 3.5 Sonnet | 长文本支持好 |
| Ollama | Qwen2.5, Llama3 | 本地部署，成本低 |

### 2.5 存储层

| 存储类型 | 技术 | 用途 |
|----------|------|------|
| 主数据库 | PostgreSQL | 结构化数据存储 |
| 缓存 | Redis | 热点数据缓存、会话存储 |
| 文件系统 | 本地存储 | PDF 文件、日志文件 |

## 4. 技术架构特点

### 4.1 异步架构

- **FastAPI**: 原生支持 async/await
- **SQLAlchemy**: 异步数据库操作
- **httpx**: 异步 HTTP 客户端
- **SSE**: 实时进度推送

### 4.2 微服务化

- **模块解耦**: 各模块独立开发部署
- **接口标准化**: RESTful API + OpenAPI
- **容器化**: Docker + Docker Compose

### 4.3 可扩展性

- **数据源扩展**: 插件式数据源接入
- **LLM 扩展**: 统一 LLM 网关接口
- **功能扩展**: 模块化业务逻辑

### 4.4 高可用性

- **健康检查**: 定期检查服务状态
- **自动重启**: 容器异常自动重启
- **负载均衡**: Nginx 反向代理
- **数据持久化**: Docker 卷持久化

## 6. 安全架构

### 6.1 网络安全

- **HTTPS**: 强制 HTTPS 访问
- **CORS**: 限制跨域请求来源
- **CSP**: 内容安全策略
- **HSTS**: HTTP 严格传输安全

### 6.2 应用安全

- **输入验证**: Pydantic 数据验证
- **SQL 注入防护**: SQLAlchemy ORM
- **XSS 防护**: 前端转义输出
- **CSRF 防护**: Token 验证

### 6.3 数据安全

- **密码加密**: bcrypt 哈希
- **API Key 保护**: 环境变量存储
- **日志脱敏**: 敏感信息不记录
- **数据备份**: 定期备份策略

## 7. 监控架构

### 7.1 应用监控

- **健康检查**: /api/v1/health
- **性能指标**: 响应时间、吞吐量
- **错误追踪**: 异常捕获和报告
- **日志收集**: 结构化日志

### 7.2 基础设施监控

- **资源使用**: CPU、内存、磁盘
- **网络流量**: 带宽使用、连接数
- **容器状态**: Docker 容器健康状态
- **数据库性能**: 查询性能、连接池

### 7.3 告警机制

- **阈值告警**: 资源使用超阈值
- **错误告警**: 应用错误率异常
- **性能告警**: 响应时间超阈值
- **可用性告警**: 服务不可用
