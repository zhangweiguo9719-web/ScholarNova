# ScholarNova 三分钟产品介绍视频稿

## 核心叙事

ScholarNova 不是“给关键词返回一串标题”的搜索页，而是一个面向复杂科研问题的论文发现、证据分析和知识沉淀工作台。演示必须同时说明产品价值、技术闭环、真实测试和公开部署方式。

## 180 秒分镜与口播

### 0–15 秒：问题与钩子

**画面**：英文深色首页，镜头从 ScholarNova 标识移动到中心搜索框。

**口播**：

> 面对一个包含研究主题、方法、数据集、时间和发表来源约束的问题，普通关键词搜索往往返回大量标题，却没有告诉研究者哪些论文真正相关、证据在哪里、下一步应该怎么研究。ScholarNova 为这个问题提供一条完整链路。

### 15–35 秒：立项依据

**画面**：比赛要求关键词依次出现：查询理解、多策略检索、综合排序、结构化整理。

**口播**：

> 项目源于科研场景下复杂学术查询的智能论文搜索与推荐赛题。我们把要求拆成四个工程目标：理解并分解复杂查询，自主选择检索策略，平衡召回率与精确率，最后把结果变成可验证、可整理的研究资产。

### 35–55 秒：研发路径

**画面**：展示 README 架构图。

**口播**：

> 系统采用前后端分离架构。查询规划器连接用户选择的大模型；检索层对接 Semantic Scholar、OpenAlex、Crossref 和 arXiv；候选论文经过去重、约束过滤和综合排序，再进入证据分析、知识库与研究路线模块。

### 55–90 秒：真实复杂检索

**画面**：进入 Search，输入“large language model traffic flow prediction”，展示 Query Plan、检索质量快照和结果数量。

**口播**：

> 以“大语言模型用于交通流预测”为例，ScholarNova 自动形成子查询并执行多源检索。页面不仅显示标题，还展示作者、摘要、年份、来源、引用量、相关度和质量信号。API 调用次数、检索轮次与端到端延时也会同步记录，便于控制比赛成本。

### 90–120 秒：论文质量与证据

**画面**：向下滚动论文卡片，停留在 Quality、Citation percentile、Citations/year 和 Quartile unverified；再打开一篇论文的 AI 分析。

**口播**：

> 质量分析使用可追溯的开放指标：当前结果集中的引用百分位、年均引用和综合质量分。对需要授权数据的 JCR 或中科院分区，系统只在来源已验证时展示，未知时明确标记为未核验，不伪造结论。AI 分析进一步提取摘要、方法、贡献、局限与证据。

### 120–145 秒：从论文到研究路线

**画面**：切换 Knowledge，展示 13 条知识条目和研究路线；展示由 SenseNova U1 生成的框架图。

**口播**：

> 有价值的论文可以直接沉淀到个人知识库。本次实测已构建 13 条知识条目，并能围绕主题归纳研究路线。SenseNova U1 专门用于把研究路线转换成框架图，让分散论文成为可继续推进的研究结构。

### 145–165 秒：比赛对标与成本

**画面**：显示基准表格，突出 0.304288 → 0.341379。

**口播**：

> 在 Asta 官方验证集的 18 条确定性子集上，系统 F1 从 0.304288 提升到 0.341379，Precision 达到 0.352313。这里明确是子集回归结果，不冒充完整比赛总分。确定性查询不调用模型，所以 Token 为零；需要大模型的查询则记录供应商返回的真实 Token。

### 165–180 秒：成熟产品与开源

**画面**：Settings 页空白 API Key 输入框、GitHub 首页和 Docker 命令。

**口播**：

> 比赛环境使用本地私有配置，公开版则采用 BYOK，仓库不包含任何密钥和授权数据。用户可以选择自己的模型和数据源，通过 Docker Compose 一键部署。ScholarNova 将检索、分析、证据和知识管理连接成了一个可以完整使用的科研产品。

## 从立项到产品研发成功的过程稿

1. **需求定义**：从赛题中提取查询理解、自主检索、综合排序、结构化展示和成本控制五类可验收目标。
2. **产品原型**：先完成搜索、论文详情和 AI 分析最小闭环，再扩展知识库与研究路线。
3. **多源接入**：统一 Semantic Scholar、OpenAlex、Crossref、arXiv 的论文结构，加入去重和 CorpusId 对齐。
4. **检索工程化**：增加规则与大模型混合查询规划、全局限流、重试、缓存、熔断和错误降级。
5. **结果可解释化**：论文卡片补齐摘要和元信息，增加相关度、引用百分位、年均引用和数据来源说明。
6. **分析可靠性**：修复模型分析连接错误，加入真实 Token 计量与规则回退，防止异常重试无上限消耗。
7. **知识沉淀**：把论文分析保存为知识条目，形成主题分类、研究路线和 SenseNova U1 框架图。
8. **官方数据测试**：获得 PaSa/Asta 授权后，将数据保留在本地，用可复现预测文件评估 Precision、Recall、F1、API 调用和延时。
9. **误差驱动优化**：逐条分析漏检与噪声，修复论文别名、BibTeX 引用、引文交集和结果预算；拒绝把验证集答案硬编码进产品。
10. **公开交付**：完成中英文与明暗主题、Docker/本地部署、空白 API 模板、CI、MIT 许可、安全说明、真实截图和公开报告。

## 拍摄检查表

- 浏览器缩放保持 100%，隐藏开发者工具、通知和个人标签页。
- 只展示已脱敏的 Settings 页面，API Key 输入框必须为空。
- 搜索页使用已经成功返回的真实结果，不在录屏时等待外部 API。
- 质量分区必须停留在 “Quartile unverified”，除非现场已接入合法授权数据。
- 指标字幕写全：“Asta 官方验证集 18 条确定性子集”，不能写成“比赛总分”。
- GitHub 展示公开 BYOK 部署；比赛私有 `.env`、授权数据和日志不出镜。

## 建议字幕

- Complex query → Query plan → Multi-source retrieval
- Evidence-rich paper cards, not title-only results
- Traceable quality signals; no fabricated quartiles
- 13-item research knowledge base
- F1 0.304288 → 0.341379 on an 18-query Asta validation subset
- Self-hosted and BYOK
