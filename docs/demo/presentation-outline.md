# ScholarAgent 答辩 PPT 大纲

## 总时长: 15 分钟

---

## 1. 项目背景与问题（2 分钟）

### 1.1 学术研究现状

**幻灯片 1: 学术研究的挑战**

- 信息爆炸：学术论文数量呈指数增长
- 检索困难：传统搜索引擎难以满足复杂查询需求
- 效率低下：研究者花费大量时间在文献检索上
- 质量参差：难以快速识别高质量、相关性强的论文

**幻灯片 2: 现有解决方案的不足**

- 关键词搜索：无法理解复杂语义
- 数据库分散：需要在多个平台重复搜索
- 人工筛选：耗时且容易遗漏
- 缺乏分析：无法提供深度分析和对比

**幻灯片 3: ScholarAgent 的定位**

- 一句话概括：面向复杂科研问题的智能检索与分析系统
- 核心价值：将自然语言查询转化为可执行检索计划
- 目标用户：科研人员、研究生、学术工作者
- 应用场景：文献综述、论文写作、研究选题、学术交流

---

## 2. 核心创新点（3 分钟）

### 2.1 复杂查询理解

**幻灯片 4: 查询理解能力**

- Level 1-5 查询复杂度支持
- 自然语言输入，自动解析约束条件
- 智能分解为多个子查询
- 示例：
  ```
  输入: "近两年有实验对比的联邦学习隐私保护方法"
  解析:
  - 时间约束: 2024-2025年
  - 内容约束: 有实验对比
  - 主题约束: 联邦学习隐私保护
  - 子查询: 3个针对性查询
  ```

**幻灯片 5: 多源聚合能力**

- 四大学术数据库集成
  - Semantic Scholar：计算机科学领域最全
  - OpenAlex：开放学术数据，覆盖范围广
  - Crossref：DOI 解析和引用数据
  - arXiv：最新预印本论文
- 并行检索，智能去重
- 结果融合和排序

**幻灯片 6: 约束验证能力**

- 硬约束满足率 ≥ 85%
- 多维度约束验证
  - 时间约束：发表日期验证
  - 内容约束：实验、方法、数据集验证
  - 来源约束：期刊、会议验证
- 不满足约束的结果自动过滤

**幻灯片 7: 证据级推荐**

- 每篇推荐附带原文证据
- 证据精确定位到论文段落
- 置信度评估
- 可追溯、可验证

**幻灯片 8: BYOK 模型网关**

- 支持多种 LLM 提供商
  - OpenAI: GPT-4o, GPT-4
  - Anthropic: Claude 3.5 Sonnet
  - 本地模型: Ollama (Qwen2.5, Llama3)
- 灵活切换，降低成本
- 用户可自定义模型配置

- **配色方案**: 蓝色系（专业、科技感）
