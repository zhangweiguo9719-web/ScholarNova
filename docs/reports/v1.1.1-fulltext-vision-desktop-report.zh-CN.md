# ScholarNova v1.1.1 全文、视觉分析与桌面版优化测试报告

日期：2026-07-19  
版本：v1.1.1

## 1. 本轮目标

本轮针对真实使用中最影响可信度和体验的三个问题进行修复：

1. AI 分析必须尽可能读取合法可用的论文全文、表格、图注和页面图像，不能把摘要分析伪装成全文分析。
2. 出版社阻止自动下载时，用户需要能够导入自己有权使用的 PDF，并在应用重启后继续使用。
3. 搜索详情侧栏需要向左拖动扩展，全文状态、图像读取数和模型 Token 消耗必须清楚可见。

## 2. 实际改动

### 2.1 全文获取与导入

- 合法开放全文解析顺序扩展为：论文直链、arXiv、OpenAlex、Unpaywall、Semantic Scholar、Crossref、会议站点、PMC。
- 下载改为流式 GET，设置 50 MB 上限，避免依赖经常被拦截的 HEAD 请求。
- 同一失败地址不会因为多个元数据服务返回相同 URL 而重复请求。
- 新增本地 PDF 导入与状态接口；文件经过 PDF 魔数、文件大小和 PyMuPDF 可解析性校验后，按论文 ID 哈希持久保存在桌面应用用户数据目录。
- 上传新 PDF 后清除旧的摘要分析缓存并重新分析。

### 2.2 全文与图像分析

- 从 PDF 提取章节化正文、表格文本、中英文“图/表、Figure/Table”图注，正文上下文上限为 48,000 字符。
- 最多渲染 3 个包含图像、矢量图或图表标题的页面，优先交给“视觉”任务配置的多模态模型。
- 若视觉模型拒绝图片输入，自动改用“分析”模型读取正文、表格与图注，并将页面图像读取数如实记为 0。
- 分析结果分别返回材料覆盖范围、材料来源、获取错误、页面图像读取数和模型是否真正完成，避免把 PDF 已解析与模型已完成混为一谈。
- 返回并显示供应商报告的输入、输出和总 Token，解决论文分析消耗长期显示为 0 的问题。

### 2.3 配置与桌面体验

- 修复重启后按任务保存的 `model_name` 没有映射到运行时模型字段的问题。
- 同一供应商的任务配置如果只设置模型、未重复填写 Key 和地址，会安全继承默认配置；不同供应商不会错误继承 Key。
- 论文详情侧栏支持鼠标向左拖动和键盘方向键调整，宽度在本机持久保存。
- Windows Portable 启动时自动创建或更新桌面快捷方式，图标和目标均指向当前版本。

测试对象：公开 PDF《Research on the Development of Moral Education Resources for Eighth-Grade Mathematics Culture Assisted by Mathematical Intelligent Agents》，DOI `10.12677/ae.2025.1581413`。

| 指标 | 结果 |
| --- | --- |
| PDF 大小 | 2,484,011 bytes |
| PDF 页数 | 7 |
| 重启后全文状态 | `available=true` |
| 文档覆盖 | `fulltext` |
| 文档来源 | `fulltext:uploaded` |
| 实际读取图表页面 | 3 |
| 模型完成 | `true` |
| 是否本地回退模板 | 否 |
| 端到端耗时 | 30.4 秒 |
| 输入 Token | 7,412 |
| 输出 Token | 777 |
| 总 Token | 8,189 |

另一次长输出验收返回 5,195 字模型分析，耗时 132.8 秒，同样确认使用持久化全文而非摘要模板。

针对用户报告的 `10.1360/ssi-2024-0050`，OpenAlex、Semantic Scholar 和 Crossref 最终均指向同一出版社地址，该地址对自动客户端返回 HTTP 418。v1.1.1 不再声称读过该全文，而是展示具体失败原因，并提供“导入全文 PDF”作为合规路径。

## 7. 使用边界

- ScholarNova 只自动获取合法开放或数据源明确提供的 PDF，不绕过付费墙、登录或出版社反爬机制。
- 用户导入 PDF 前应确认自己拥有访问和分析权限。
- 真正读取页面图像需要在设置中为“视觉”任务配置支持图片输入的多模态模型；文本模型会自动降级为正文、表格和图注分析，并明确标注降级结果。
- 期刊 JCR/中科院分区仍需用户导入有权使用的授权数据，系统不会伪造分区。
