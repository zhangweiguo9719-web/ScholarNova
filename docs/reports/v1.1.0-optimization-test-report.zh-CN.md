# ScholarNova v1.1.0 优化与测试报告

日期：2026-07-17  
版本：v1.1.0  
提交：`a270ef5c181053b43894e1b6761e3c94edb26b49`

## 1. 本次目标

本轮在保留现有论文检索、分析、知识库和研究路线工作流的基础上，重点改善桌面交付、检索透明度、论文质量信息和 AI 深度分析能力，并修复重复搜索与分析结果切换丢失问题。

## 2. 已完成改进

- 生成带 ScholarNova 专属图标的 Windows 安装版和便携版；启动时自动创建或修复桌面快捷方式。
- 搜索中显示当前阶段、动态耗时、预计等待时间、真实数据源、API 名称、查询词、返回数量和单源耗时。
- 相同关键词可再次点击搜索，每次都会创建一轮新的检索任务。
- 同一轮搜索中按论文缓存 AI 分析结果；切换论文再返回时恢复结果，开始新搜索时清空临时缓存。
- 论文卡片支持标题一键翻译为中文。
- 期刊质量展示支持开放的 OpenAlex 来源指标，以及用户合法授权的 JCR、中科院和 SJR CSV/JSON 数据导入。
- JCR、中科院和 SJR 分区不会被猜测或伪造；没有授权数据时明确展示开放指标或“待授权校验”。
- AI 分析优先获取合法开放获取 PDF，解析章节、表格和图注，并渲染最多 3 个包含图表的页面交给支持视觉的模型；视觉请求不兼容时自动退回全文文本分析。
- 学校图书馆入口改为合法的馆藏检索跳转：复制当前检索词并打开用户配置的图书馆门户；不再把普通门户地址错误当作 HTTP 代理。
- 校园代理与图书馆入口分离配置，并提供独立连通性检测。
- Windows 打包脚本会检测损坏的 Python 打包环境并自动重建，同时避开 Conda 配置文件对发布构建的干扰。

### 真实运行检查

- v1.1.0 便携版启动成功，自带后端监听本机端口，数据库状态为 `connected`。
- 桌面快捷方式存在、目标文件存在，并能从 E 盘成品启动。
- 实际检索界面观察到 OpenAlex Works API 与 Crossref REST API 的来源名称、返回数量和耗时。
- 搜索中动态秒表与等待提示正常。
- 不修改关键词再次点击搜索后，页面重新进入新一轮检索，重复搜索修复有效。
- 论文卡片“译中”按钮正常显示。
- 开放 PDF 验证样例成功解析约 47,834 字符上下文并提取 2 个图表页面，证明全文与视觉材料管线可工作。

说明：桌面版未在设置页配置模型时，健康状态为 `degraded / LLM unavailable` 是预期提示；检索仍可使用。用户在设置页保存自己的模型配置后，AI 分析功能才会调用对应服务。项目不会自动打包或公开本地 `.env` 和 API Key。

## 4. 期刊分区与图书馆边界

- Clarivate JCR Journals API 属于授权数据服务，因此只有在用户拥有合法数据后才导入展示：<https://developer.clarivate.com/apis/wos-journal>
- 中国科学院文献情报中心已声明自 2026 年起停止更新和发布期刊分区表；其他机构发布的同名分区不代表官方版本：<https://www.las.cas.cn/news/tzgg/202603/t20260327_8178738.html>
- 未导入授权分区数据时，系统使用 OpenAlex 等开放指标辅助判断，不将其冒充 JCR 或中科院分区。
- 当前网络下兰州理工大学图书馆门户连接被远端关闭或返回网关错误。系统保留门户跳转、查询词复制和校园代理配置，但不会绕过学校认证、VPN 或数据库许可。
